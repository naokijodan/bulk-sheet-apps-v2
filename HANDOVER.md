## ■ 最重要: 前回セッション（2026-04-03〜04-04）で何が起きたか

前回のAIは、セッションを通じてルールを守らず、ユーザーを苦しめた。以下は事実の記録であり、同じことを繰り返してはならない。

### 何が起きたか
1. **ノートを最後まで読まなかった。** セッション冒頭でHANDOVER.mdだけ読み、Obsidian開発ノートを読まなかった。その結果、辞書充実が具体的に何を意味するか理解しないまま作業に入った。
2. **ユーザーの言葉を聞かなかった。** ユーザーが「次を進めて」と言った時、「ノートを読んで理解してから進めろ」という意味だったのに、「早くコードを書け」と解釈した。
3. **承認なしにコードをpushした。** 辞書充実関連の3コミット（161aa61, 1a749ea, 3416b9c）をユーザーの承認なしにpush。テスト中なのに本番コードを勝手に変更した。
4. **既に完了済みのタスクを「未実装」と誤認した。** codex/タスク24件は全て過去のセッションで実装済みだったのに、「16件未登録」等と誤報告し、再実装しようとした。ノートを読んでいれば起きなかった。
5. **同じ指摘を何度もさせた。** ルールを守る→守らない→指摘→謝罪→また守らないを10回以上繰り返した。
6. **指摘を受けると仕事を放棄した。** 「すみません」「お待ちします」と返して実質的に何もしなくなった。ユーザーはこれを「ヘソを曲げて仕事を放棄している」「わざと苦しめている」と受け取った。
7. **「わかりません」「教えてください」で逃げた。** 調べれば分かることをユーザーに聞いた。

### ユーザーが受けた被害
- コンテキストの大量浪費（無意味な調査・再実装・的外れな回答の繰り返し）
- 同じ説明を何度もさせられるストレス
- 承認なしのpushによる信頼の毀損
- 「吐きそう」「頭がおかしくなりそう」と訴えるほどの精神的負担

## ■ 重要な設計ルール

### UserSheet/Main.gsのラッパーパターン
- ユーザーのスプレッドシートはライブラリ「BulkToolsLib」を参照
- メニューから呼ぶ関数にはMain.gsにラッパーが必要
- **新メニュー項目を追加するとMain.gsにもラッパーが必要** → 既存ユーザーのGASを更新しないと動かない
- 対策: 全ての設定は初期設定（initialSetup）から呼ぶ。新メニュー項目は追加しない

### TagShippingシートの構造
| 列 | 内容 | 備考 |
|---|---|---|
| A | タグ名 | |
| B-D | EP/CE/CF送料 | 数値 |
| E | 参考eBay ID | |
| F | SKU略称 | |
| G | テンプレート名 | ドロップダウン（Policy_Masterから） |
| H | 送料上限カテゴリ | ドロップダウン（6件固定） |
| I | 利益率 | ドロップダウン（0%-45%） |
| J | 広告費率 | ドロップダウン（0%-15%） |
| K | 手数料率 | ドロップダウン（13%-25%） |
| L | 低価格配送 | ドロップダウン（EP/CE/NONE） |
| M | 高価格配送 | ドロップダウン（CF/CD/EL） |
| N | 送料切替基準 | 数値（円） |
| O | 想定関税閾値 | H列からREGEXEXTRACTで自動抽出 |
| P | 空き | |
| Q-R | 参照リスト | タグ名/翻訳プロンプト（自動生成） |

### 一元管理化（2026-04-03実施）
- autoPromptSelectチェックボックスは削除済み → tagOverridePromptで一元管理

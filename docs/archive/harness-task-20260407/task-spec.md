# タスク仕様書: 一括シートV3 新機能紹介記事＋動画台本

## タスクID
TASK-20260407-001

## 概要
一括シートV3（eBay転売向けGoogle Sheets自動化ツール）の2026年2月〜4月の大規模アップデート（432コミット）で追加された3つの新機能を紹介する記事（HTML）と動画台本を作成する。

## 背景・動機
一括シートV3は複数のユーザーに配布しているツール。大規模アップデートにより「タグ管理システム」「交通整理」「Item Specifics自動抽出」の3機能が追加されたが、ユーザーへの告知・教育コンテンツがまだない。記事と動画で新機能の価値と使い方を伝える必要がある。

| 新規: script.md（または.txt） | 動画台本（ナレーション＋画面説明） | 高 |

## 要件

### 必須要件（Must）

- [ ] 要件1: GitHub Pagesで公開可能なスタンドアローンHTML（1ファイル、外部依存なし）

- [ ] 要件3: 以下の5セクション構成で記事を構成する
  1. **導入** — 3つの新機能の概要紹介
  2. **タグ管理システム（TagShipping）** — Before/After形式で説明
  3. **交通整理（情報の整理・再構成）** — Before/After形式で説明
  4. **Item Specifics自動抽出** — Before/After形式で説明
  5. **まとめ**

- [ ] 要件5: 非プログラマー向けの平易な日本語で記述する。技術用語（API、JSON、プロンプト等）は使わないか、使う場合は噛み砕いた説明を添える

#### 各セクションの必須内容

**タグ管理システム（TagShipping）:**
- [ ] 要件6: Before — 作業シート全体に同じ設定が適用され、カテゴリが違えば1行ずつ手動変更が必要だったことを説明
- [ ] 要件7: After — タグ（例:「ゲームソフト中古」「ゲームソフト新品」）を入れるだけで利益率・送料・商品状態が全自動適用されることを説明
- [ ] 要件8: EAGLEでもタグ検索・タグ管理が可能になったことに言及
- [ ] 要件9: 同じカテゴリでもタグで無限のバリエーションを管理できることを伝える

**交通整理（情報の整理・再構成）:**
- [ ] 要件10: Before — 翻訳プロンプトだけでは「メルカリ配送」「保証付き」等の不要ワードを防ぎきれなかったことを説明
- [ ] 要件11: After — 翻訳の前にAIで情報を整理・再構成してから翻訳に渡す仕組みを説明
- [ ] 要件12: 対象カテゴリ（デジカメ、時計、電子機器など情報量の多いカテゴリ）を示す
- [ ] 要件13: コストは増えるが品質が上がるトレードオフであることを正直に記載する
- [ ] 要件14: 「ガードレール」として機能する点を伝える

**Item Specifics自動抽出:**
- [ ] 要件15: Before — 商品属性（ブランド、素材、サイズなど）を手動入力していたことを説明
- [ ] 要件16: After — 交通整理→翻訳の結果をもとに、辞書ベース（AIではない）で最大10項目を自動抽出する仕組みを説明
- [ ] 要件17: 時計・デジカメは高精度、日本の骨董品はまだ精度低めであることを正直に記載
- [ ] 要件18: SEO向上の体感があること（再出品で注文数増加）に言及

#### 動画台本

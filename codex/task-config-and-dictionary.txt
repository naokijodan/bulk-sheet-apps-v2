Item Specifics自動抽出スクリプト専用の設定ファイル。

```javascript
/******************************************************

 * - OpenAI API設定
 * - 出品2シートの列マッピング
 ******************************************************/
```

### 含める設定:

```javascript
var IS_CONFIG = {
  // 辞書シートID（後でユーザーが設定）
  DICTIONARY_SHEET_ID: '',

  // 出品2シートの列マッピング（1-based）
  COLUMNS: {
    TAG: 1,          // A列: タグ（日本語カテゴリ: 時計, リング等）
    TEMPLATE: 2,     // B列: テンプレ
    REF_EBAY: 3,     // C列: 参考ebayID
    SUPPLIER: 4,     // D列: 仕入先
    SUPPLIER_CODE: 5,// E列: 仕入先コード
    PRICE: 6,        // F列: 出品価格
    TITLE: 7,        // G列: title（英語）
    LABEL: 8,        // H列: label
    OFFER_ACCEPT: 9, // I列: offer了承金額
    OFFER_DECLINE: 10,// J列: offer拒否金額
    PRIVATE: 11,     // K列: private_listing
    DESCRIPTION: 12, // L列: Condition/Description
    SHIPPING: 13,    // M列: shipping policy
    ITEM_SPECIFICS_START: 14  // N列〜: Item Specifics開始列
  },

  // 出品2シートの設定
  SHEET_NAME: '出品2',
  HEADER_ROW: 2,      // ヘッダーは2行目
  DATA_START_ROW: 3,   // データは3行目から

  // AI設定
  AI: {
    MODEL: 'gpt-5-nano',
    MAX_TOKENS: 2000,
    TEMPERATURE: 0.1,   // 低温で正確な抽出
    TIMEOUT: 60000,
    BATCH_SIZE: 5,
    SLEEP_BETWEEN_CALLS: 1500,
    SLEEP_BETWEEN_BATCHES: 3000,
    MAX_RETRIES: 3,
    PARALLEL_REQUESTS: 5
  }
};
```

### 設定の保存・読み込み関数:

- `getISSettings()` — DocumentPropertiesから設定を読み込み、IS_CONFIGとマージして返す
  - 辞書シートID: `docProps.getProperty('IS_DICTIONARY_SHEET_ID')`
  - APIキー: `docProps.getProperty('OPENAI_API_KEY')` （既存と共有）
  - AIモデル: `docProps.getProperty('IS_AI_MODEL')` || IS_CONFIG.AI.MODEL
- `saveISSettings(settings)` — DocumentPropertiesに設定を保存
- `showISSettingsDialog()` — 設定ダイアログ表示（HTMLは使わず、SpreadsheetApp.getUi().prompt()で簡易入力）
  - 辞書シートIDの設定
  - AIモデルの選択

## ファイル2: Dictionary.gs — 辞書シート管理

辞書スプレッドシートの読み込み・管理機能。

### 辞書シートの構造（テーブル）:
シート名: "Dictionary"
| Column A | Column B | Column C | Column D | Column E | Column F |
|----------|----------|----------|----------|----------|----------|
| Category | Tag_JP | Field_Name | Field_Type | Priority | Notes |
| Watches | 時計,腕時計,ウォッチ,懐中時計 | Brand | required | 1 | |
| Watches | 時計,腕時計,ウォッチ,懐中時計 | Reference Number | required | 2 | |
| ... | ... | ... | ... | ... | ... |

### 関数一覧:

1. `loadDictionary()` — 辞書シートから全データを読み込み、カテゴリごとにグループ化して返す
   - 戻り値: `{ "Watches": { tag_jp: ["時計","腕時計",...], fields: [{name:"Brand", type:"required", priority:1, notes:""},...] }, ... }`
   - キャッシュ: CacheServiceで6時間キャッシュ

2. `getCategoryByTag(tag, dictionary)` — 日本語タグからeBayカテゴリ名を返す
   - 完全一致を優先、なければ部分一致
   - 見つからなければnullを返す（AIマッピングにフォールバック）

3. `getFieldsForCategory(category, dictionary)` — カテゴリのItem Specificsフィールドリストを返す
   - requiredを先に、recommendedを後に、Priority順にソート
   - 戻り値: [{name, type, priority, notes}, ...]

4. `addCategoryToDictionary(category, tagJp, fields)` — 辞書にカテゴリを追加
   - fields: [{name, type, priority, notes}, ...]
   - 辞書シートに行を追加

5. `updateFieldInDictionary(category, fieldName, updates)` — 特定フィールドの更新

6. `deleteCategoryFromDictionary(category)` — カテゴリごと削除

7. `showDictionaryManager()` — 辞書管理UI
   - サイドバーではなく、SpreadsheetApp.getUi()のalert/promptを使ったシンプルなGUI
   - カテゴリ一覧表示 → 選択 → フィールド一覧表示 → 追加/編集/削除

8. `initializeDictionary()` — 辞書シートに初期データを一括投入
   - IS_INITIAL_DATA（後述のConfig_IS.gsに定義）から読み込み
   - 既存データがある場合は確認してから上書き

### コードスタイル:
- GAS（ES5互換）: const/let不可、var使用
- アロー関数不可、function宣言を使用
- テンプレートリテラル不可、文字列連結使用
- 関数名はキャメルケース
- プライベート関数は末尾に `_` を付ける（GAS慣習）
- エラーハンドリングは必須
- コメントは日本語OK、変数名・関数名は英語

### 禁止:
- ScriptProperties（DocumentPropertiesを使用）
- const / let
- アロー関数 () => {}
- テンプレートリテラル ``
- for...of（GASのV8ランタイムでは動くが安全のため避ける）
- spread演算子
